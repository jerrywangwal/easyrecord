项目介绍：EasyRecord 是什么？

🎯 核心定位

EasyRecord 是一个运行在浏览器里的白板录屏工具，专为教学、演示、内容创作设计。用户可以在画布上实时绘图，同时录制屏幕 + 摄像头 + 麦克风，最终导出 MP4 视频。

🖊️ 手绘风格白板 📹 画中画摄像头 🎙️ 麦克风录音 📦 MP4 导出 🗂️ 多页幻灯片 ↩️ 撤销/重做

✅ 技术选型原则

  • 纯前端，无需后端服务器
  • 零框架，原生 HTML/CSS/JS
  • 依赖最少化，只引入两个库
  • 性能优先，60fps 流畅绘图

📦 仅一个外部依赖

  • Rough.js — 手绘风格图形渲染

包括 MP4 录制在内，其余所有功能全部用原生 Web API 实现，无额外依赖。

├── app.js       ← 核心逻辑：绘图、录制、幻灯片管理

app.js所有交互逻辑，包裹在 IIFE 中避免污染全局~1280 行
server.py带 COOP/COEP 头的开发服务器，供 ffmpeg.wasm 使用~26 行
💡
为什么 app.js 用 IIFE 包裹？ 所有代码都包在 (function() { 'use strict'; ... })() 里，这样内部变量不会泄露到 window 全局对象，避免与引入的库冲突。这是不用打包工具时的最佳实践。

画布系统：Canvas 核心原理

🖼️ 状态驱动渲染

所有绘图元素都保存在一个 状态对象（state） 里，每次有任何变化，都完整重绘整个画布。这是最简单、最可靠的渲染方式。

app.js — 状态对象
const state = {
  tool: 'select',       // 当前工具
  color: '#1e1e2e',    // 当前颜色
  strokeSize: 2,      // 线宽
  roughStyle: true,   // 手绘风格开关
  slides: [{ elements: [], bgColor: '#ffffff', undoStack: [] }],
  currentSlideIdx: 0, // 当前页索引
  isRecording: false, // 录制中？
  webcamStream: null, // 摄像头流
  // ... 更多属性
};

📐 自适应比例缩放

画布会根据窗口大小和用户选择的比例（16:9、9:16、3:4、1:1）自动计算尺寸，保证内容不变形。

app.js — resizeCanvas()
function resizeCanvas() {
  const ratios = { '16:9': 16/9, '9:16': 9/16, '3:4': 3/4, '1:1': 1 };
  const ratio = ratios[state.canvasRatio];

  let w = availW;
  let h = w / ratio;
  // 如果高度超出，以高度为准
  if (h > availH) { h = availH; w = h * ratio; }

  canvas.width = Math.floor(w);
  canvas.height = Math.floor(h);
}
💡
关键技巧：先算宽，再算高，超出就反过来 这个双向约束算法保证画布始终适应容器，无论横屏竖屏都不会溢出。
3
绘图工具系统

🖊️ 自由画笔

鼠标按下时记录路径点数组，松开时保存。绘制时用 二次贝塞尔曲线（quadraticCurveTo）平滑连接各点，效果比直线连接自然。

📐 几何图形

按下记录起点，拖动时实时预览（重绘 + 临时元素），松开时将形状保存进 elements 数组。支持矩形、椭圆、菱形、直线、箭头。

✏️ 手绘风格

几何图形通过 Rough.js 渲染，会自动添加轻微的不规则抖动，让线条看起来像手画的。可以在设置中关闭，切换为精确几何模式。

🔤 文字工具

点击画布弹出一个透明的 textarea 覆盖层，输入完成后把文字渲染到 Canvas 上，字体用 Caveat（手写体），支持多行。

工具快捷键数据结构
选择/移动V通过 boundingBox 命中测试
画笔P{ type:'pen', points:[[x,y],...] }
荧光笔H同上，透明度 35%，线宽 ×8
橡皮擦E按中心点距离过滤删除元素
直线/箭头L / A{ type:'arrow', x1,y1,x2,y2 }
矩形R{ type:'rect', x1,y1,x2,y2 }
椭圆O{ type:'ellipse', x1,y1,x2,y2 }
文字T{ type:'text', x,y,text,fontSize }

↩️ 撤销系统（Undo Stack）

每次开始绘制前，把当前 elements 数组的深拷贝压入 undoStack。按 Ctrl+Z 时，弹出最近一次状态恢复。最多保存 50 步，避免内存占用过大。

摄像头画中画（PiP）

📹 getUserMedia API

通过浏览器的 navigator.mediaDevices.getUserMedia() 获取摄像头流，将视频流赋给 <video> 元素自动播放（静音、无控件）。

app.js — 启动摄像头
async function startWebcam() {
  const stream = await navigator.mediaDevices.getUserMedia({
    video: { facingMode: 'user', width: 320, height: 320 },
    audio: false  // 视频只需画面，音频单独录制
  });
  webcamVideo.srcObject = stream;
  state.webcamStream = stream;
}

🔲 圆形裁切 + 定位修复

PiP 窗口用 border-radius: 50% + overflow: hidden 实现圆形。
关键修复：将 PiP 元素从 canvasWrapper（有 overflow:hidden）移出到 canvasArea，避免切换画布比例时被裁切。

⚠️
常见坑：overflow:hidden 导致 PiP 被遮挡 如果 PiP 放在有 overflow:hidden 的容器里，当容器变窄（如切换 3:4 比例时），PiP 会被直接裁掉。解决方案：把 PiP 放到更外层的容器（canvasArea），让 canvasArea 做 position:relative 参照。

🖱️ 可拖动实现

监听 mousedown/mousemove/mouseup 三个事件，记录拖动起始位置和元素起始位置，计算偏移量更新 left/top，边界约束在 canvasArea 范围内。

录制系统：MediaRecorder API

🎬 录制原理

浏览器提供 MediaRecorder API，可以录制任意 MediaStream。核心挑战是：如何同时录制画布内容 + 摄像头画面 + 麦克风？

canvas.captureStream(30fps)
+
webcamVideo
→
合成 Canvas (offscreen)
+
麦克风音频
→
MediaRecorder → WebM
💡
为什么需要合成 Canvas？ 如果直接录制主画布，摄像头 PiP 是一个独立的 HTML 元素，不在 Canvas 里，录制结果里不会有摄像头。所以需要创建一个离屏 Canvas，每帧把主画布内容 + 摄像头视频手动合成进去。
app.js — 合成每一帧
function drawFrame() {
  if (!state.isRecording) return;

  // 1. 把主画布内容复制进合成画布
  compCtx.drawImage(canvas, 0, 0);

  // 2. 把摄像头圆形画进右下角
  compCtx.save();
  compCtx.beginPath();
  compCtx.arc(cx, cy, radius, 0, Math.PI * 2);
  compCtx.clip();                  // 圆形裁切
  compCtx.scale(-1, 1);             // 左右镜像
  compCtx.drawImage(webcamVideo, ...); // 画摄像头
  compCtx.restore();

  requestAnimationFrame(drawFrame); // 循环，每帧执行
}

⏱️ 录制计时器

用 setInterval 每秒更新，计算 Date.now() - startTime 的差值，格式化为 MM:SS 显示在屏幕上方。

🎙️ 麦克风权限

录制开始时单独申请麦克风权限（audio: true），失败了也没关系，降级为无声录制，不影响画面录制。

MP4 导出：浏览器原生方案

💡 从 ffmpeg.wasm 到原生录制

最初方案是录制 WebM 再用 ffmpeg.wasm 在浏览器端转码为 MP4。但 ffmpeg.wasm 需要加载约 20MB 的 wasm 文件，国内网络访问 CDN 不稳定，导致转码经常失败、直接 fallback 成 WebM。

现在的方案更简单：让 MediaRecorder 直接录 MP4。Chrome 130+、Safari、新版 Edge 都已原生支持 video/mp4 录制，录完即下载，零转码、零等待、零网络依赖。

检测支持的格式
→
MediaRecorder 直录 MP4
→
直接下载 .mp4
app.js — 格式优先级检测
function getSupportedMimeType() {
  // 优先选 MP4，浏览器原生支持则直接录制，无需转码
  const types = [
    'video/mp4;codecs=avc1,mp4a.40.2',  // H.264 + AAC（最佳兼容性）
    'video/mp4;codecs=avc1',
    'video/mp4',
    'video/webm;codecs=vp9,opus',       // 旧版 Chrome 的 fallback
    'video/webm',
  ];
  for (const t of types) {
    if (MediaRecorder.isTypeSupported(t)) return t;
  }
  return '';
}
app.js — 下载逻辑
async function downloadRecording() {
  const mimeType = state.mediaRecorder.mimeType;
  const isNativeMp4 = mimeType.includes('mp4');

  if (isNativeMp4) {
    // 原生 MP4 — 直接下载，无需任何转码
    const blob = new Blob(state.recordedChunks, { type: 'video/mp4' });
    triggerDownload(blob, `easyrecord-${Date.now()}.mp4`);
    showToast('✅ MP4 视频已下载！', 2800);
    return;
  }
  // WebM fallback（旧浏览器）→ 尝试 ffmpeg.wasm 转码，失败则直接给 WebM
}
✅
兼容性覆盖 Chrome 130+、Safari（全版本）、Edge 130+ 均支持原生 MP4 录制，覆盖绝大多数用户场景。旧版 Chrome 会 fallback 到 WebM，并提示用户升级浏览器。

多页幻灯片系统

🗂️ 数据结构设计

每个 Slide 是一个独立的对象，包含自己的元素列表、背景色和撤销栈。所有 Slide 保存在 state.slides 数组里。

Slide 数据结构
// 每个 slide 是独立的数据孤岛
state.slides = [
  {
    elements: [ /* 该页的所有绘图元素 */ ],
    bgColor: '#ffffff',
    undoStack: [ /* 该页的撤销历史 */ ]
  },
  // ... 更多页
];

🖼️ 缩略图渲染

每个 Slide 缩略图是一个小 Canvas，切换/修改时在离屏 Canvas 上缩放重绘所有元素，再复制到缩略图 Canvas。

🔀 拖拽排序

利用 HTML5 原生 Drag & Drop API，监听 dragstart / dragover / drop 事件，实现 Slide 拖拽重排序。

⌨️ 键盘切换

按方向键 ← → 可以快速切换 Slide，方便演示时操作。切换时有滑入动画效果。

🚀 Vercel 部署步骤

1️⃣

创建 vercel.json（设置响应头）

在项目根目录创建此文件，告诉 Vercel 为所有响应添加 COOP/COEP 头，让 ffmpeg.wasm 可以正常工作。

2️⃣

安装 Vercel CLI

在终端运行：npm install -g vercel

3️⃣

登录并部署

运行 vercel login 用 GitHub 账号登录，然后 vercel --prod 一键部署，完成后获得 xxx.vercel.app 链接。

vercel.json — 响应头配置
{
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        { "key": "Cross-Origin-Opener-Policy", "value": "same-origin" },
        { "key": "Cross-Origin-Embedder-Policy", "value": "require-corp" },
        { "key": "Cross-Origin-Resource-Policy", "value": "cross-origin" }
      ]
    }
  ]
}

🔄 修改代码后如何更新上线？

每次修改完代码，在终端运行以下命令即可一键重新部署，几秒内生效：

终端 — 重新部署命令
# 设置临时环境变量（解决部分网络 SSL 证书问题）
export NODE_TLS_REJECT_UNAUTHORIZED=0

# 进入项目目录
cd /path/to/your/project

# 部署到生产环境
vercel --prod
